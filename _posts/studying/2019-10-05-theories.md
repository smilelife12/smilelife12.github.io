## HTTPS와 SSL 인증서

### HTTPS VS HTTP
- - -
HyperText Transfer Protocol의 약자로, HTML을 전송하기 위한 통신규약이다. HTTPS에서 마지막 S는 Over Secure Socket Layer의 약자로 Secrue에서 알 수 있듯이 보안이 강화된 HTTP이다. HTTP는 암호화되지 않은 방법으로 전송하기 때문에 감청이 쉽다. 이를 보안한 것이 HTTPS이다.

### HTTPS와 SSL
- - -
HTTPS와 SSL을 같은 의미로 이해하는 경우가 많다. 이것은 정확하지 않은데, HTTPS도 SSL 프로토콜 위에서 돌아가는 프로토콜이다.  


### SSL과 TLS
- - -
둘은 같은 말이며 네스케이프에서 SSL이 발명되었고, 표준화 기구인 IETF의 관리로 변경되며 TLS라는 이름으로 바뀌었다.  


### SSL 디지털 인증서
- - -
클라이언트와 서버간의 통신을 제3자가 보증해주는 전자화된 문서다. 클라이언트가 서버에 접속한 직후에 서버는 클라이언트에 인증서 정보를 전달한다. 신뢰할 수 있는 것인지 검증한 후 다음 절차를 수행한다. 이를 이용할 때 장점은 다음과 같다.
- 통신내용이 공격자에게 노출되는 것을 막을 수 있다.
- 클라이언트가 접속하려는 서버가 신뢰할 수 있는 서버인지 판단할 수 있다..
- 통신 내용의 악의적인 변경을 방지할 수 있다.  


### SSL에게 사용하는 암호화의 종류
- - -

#### 대칭키
암호화를 할 때 사용하는 일종의 비밀번호를 키(Key)라고 한다. 암호를 푸는 행위를 복호화라 한다. 대칭키는 공통의 키를 공유해 암호화된 자료를 복호화 할 때 대칭키를 입력하는 것이다.  


#### 공개키
대칭키의 단점은 대칭키를 전달하는 것이 어렵다는 단점이 있다. 이 대칭키가 유출되면 복호화가 되기 때문에 암호화가 의미가 없다. 이에 대한 대안으로 나온게 공개키이다.  
A키로 암호화를 하면 B키로 복호화하고, B키로 암호화하면 A키로 복호화한다. 두 개의 키중 하나를 비공개키(private key, 개인키, 비밀키)로 하고, 하나를 공개키(public key)로 지정한다. 비공개키를 자신이 갖고, 공개키를 공유한다. 공개키를 이용해 암호화를 하고, 이 암호화 된 것을 비공개키를 이용해 정보를 복호화 한다. 비공개키를 모르면 복호화가 불가능하다.  
응용을 하면, 비공개키를 이용해 정보를 암호화 하고, 공개키와 함께 전송한다. 그러면 정보와 공개키를 이용해 암호화된 정보를 복호화한다. 이 과정에서 공개키가 유출되면 위험할 수 있지만, 데이터 보호가 목적이 아니기 때문에 상관이 없다. 이는 공개키를 통해 복호화를 할 수 있다는 것은 데이터를 비공개키에 의해 암호화 되었다는 것을 의미하기 때문에 신원을 보장한다. 이를 전자 서명이라 부른다.    


### SSL 인증서
- - -
핵심 요소
1. 클라이언트가 접속한 서버가 신뢰 할 수 있는 서버임을 보장한다.
2. SSL 통신에 사용할 공개키를 클라이언트에게 제공한다.  


#### CA
접속한 서버가 의도한 서버가 맞는지 보장하는 역할을 하는 민간기업을 부르는 명칭이다.


#### 사설 인증기관
자신이 직접 보증하는 역할을 할 수 있지만 공인된것이 아니므로 부라우저에 경고가 뜬다.  


#### SSL 인증서 내용
1. 서비스의 정보(인증서 발급한 CA, 서비스 도메인 등등)
2. 서버 측 공개키(공개키 내용, 공개키 암호화 방법)  
내용은 크게 위의 두개로 나뉜다. 위와 같은 내용은 CA에 의해서 암호화 된다. 이 때 사용하는 기법이 공개키 방식이다. CA는 자신의 비공개키를 이용해 서버가 제출한 인증서를 암호화한다.  


#### CA를 브라우저는 안다.
브라우저는 미리 CA의 리스트를 포함해서 그 공개키를 알고 있다.


### SSL의 동작방법
- - -
공개키와 대칭키를 혼합해서 사용한다. 클라이언트와 서버가 주고 받는 실제정보는 대칭키 방식으로 암호화하고, 대칭키 방식으로 암호화된 실제 정보를 복호화할 때 사용할 대칭키는 공개키 방식으로 암호화해서 클라이언트와 서버가 주고 받는다.  
네트워크를 이용해 통신하는 내부적인 3가지 단계는  
`악수-> 전송 -> 세션 종료`  
이것이 사용자에게 노출은 되지 않는다.  


#### 1. 악수(handshake)  
클라이언트와 서버사이에 인사같은 개념이다. 이를 SSL 방식에서도 브라우저와 서버가 핸드쉐이크를 한다.  
1. 클라이언트가 서버에 접속하고 이 단계를 Client Hello라고 한다. 이때 주고받는 데이터는
  - 클라이언트 측에서 생성한 랜덤 데이터
  - 클라이언트가 지원하는 암호화 방식
  - 세션 아이디
2. 서버는 Client Hello에 대한 응답으로 Server Hello를 하게 된다. 이때 주고받는 데이터는
  - 서버 측에서 생성한 랜덤 데이터
  - 서버가 선택한 클라이언트의 암호화 방식
  - 인증서
3. 클라이언트에서 서버의 인증서가 CA의 것인지 확인하기 위해 내장된 CA리스트를 확인한다. 없으면 경고를 출력한다. 그리고 발급을 확인하기 위해 CA의 공개키를 이용해 인증서를 복호화하고 성공하면 인증된 것이다. 이제 생성된 랜덤데이터 두개를 조합해 premaster secret이라는 키를 생성한다. 이 때 사용할 기법은 대칭키이므로 이 값은 노출되어선 안된다. 이 값을 전달할 방법으로 공개 키 방식을 쓴다. 인증서 내의 공개키를 이용해 암호화해서 전달한다.
4. 서버는 pre master secret 값을 자신의 비공개키로 복호화 하고 두 값을 공유했으므로 이 과정을 거쳐 master secret을 만든다.이를 통해 세션 키를 생성한다. 이를 대칭키 방식을 이용해 주고 받는다.
5. 클라이언트와 서버는 핸드쉐이크 종료를 서로에게 알린다.  

#### 2. 세션
서버와 클라이언트가 데이터를 주고받는 단계이다. 정보를 상대방에게 전송하기전 session key 값으로 대칭키 방식으로 암호화한다.  


#### 3. 세션종료
서로에게 알린후 세션키를 삭제한다.

## 탐색 알고리즘
- - -

### 선형 검색(Linear Search)
- 배열의 좌측부터 시작해 찾는 값과 하나씩 배열의 각 요소와 비교
- 찾으려는 값을 발견하면 배열의 해당 인덱스를 반환
- 찾으려는 값이 없으면 -1을 반환
- 시간 복잡도 `O(n)`
- 거의 사용되지 않음
```python
def linear_search(arr, n, x):
  for i in range(n):
    if arr[i] == x:
      return i;
  return -1
```

### 이진 검색(Binary Search)
- 배열을 반으로 나누어 반복
- 정렬된 배열에서 사용
- 중간 값과 비교하여 그 값이 더 작ㄷ으면 우측, 크면 좌측으로 이동해 반복
- 시간 복잡도 `O(log n)`
```python
def binary_search(arr, l, r, x):
  if r >= l:
    mid = l + (r-l)//2
    if arr[mid] == x:
      return mid
    if arr[mid] > x:
      return binary_search(arr, l, mid-1, x)

    return binary_search(arr, mid+1, r, x)
  return -1
```


### 이진 탐색 트리(BST, Binary Search Tree)
- 이진 트리로 만들어지며 조건은
  1. 모든 노드는 키 값을 갖고, 같은 키는 없음
  2. 왼쪽 부속 트리의 모든 노드의 키 값은 루트 키 값보다 작음
  3. 오른쪽 부속 트리의 모든 노드의 키 값은 루트의 키 값보다 큼
  4. 왼쪽과 오른쪽 부속 트리도 이진 탐색 트리  
- 탐색, 삽입, 삭제 시간이 트리 높이만큼 걸린다.
- 시간 복잡도 `O(log n)`


## 정렬 알고리즘
- - -
### 선택 정렬(Selection Sort)
- 전체 원소들 중 최소값을 찾아 처음 원소에 놓고, 나머지 값 중 최소값을 찾아 다음에 놓고 이를 반복한다.
```python
def selection_sort(arr, n):
  for i in range(n):
    min = i
    for j in range(i+1, n):
      if arr[j] < arr[min]:
        min = j
      arr[min], arr[j] = arr[j], arr[min]
```
- 최적 시간 복잡도 `O(n^2)`, 평균 시간 복잡도`O(n^2)`, 최악 시간 복잡도`O(n^2)`


### 삽입 정렬(Insertion Sort)
- - -
- 이미 순서화된 파일에 새로운 하나의 값을 넣어 정렬하는 개념이다.
- 첫 번째 값은 정렬이 된 것으로 가정하고, 두 번째 값부터 비교하여 더 작은 값이라면, 교환한다. 그 다음 두 번째까지 정렬되었다 가정하고, 세 번째 값을 두 번째 값까지 탐색하여 자리를 찾아 집어넣는다.
```python
def insertion_sort(arr, n):
  for i in range(1,n):
    key = arr[i]
    for j in range(i-1, 0, -1):
      if key > arr[j]:
        break
      arr[j+1] = arr[j]
    arr[j+1] = key
```
